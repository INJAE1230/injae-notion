## 1. 시작하기

### 최초 설치
```bash
git clone https://github.com/INJAE1230/chungchosu-notion.git
cd chungchosu-notion
npm install
```

### 환경 변수 설정
프로젝트 루트에 `.env` 파일을 만들고 아래 내용을 입력:
```
NOTION_TOKEN=노션_API_토큰
NOTION_DATABASE_ID=노션_데이터베이스_ID
OPENAI_API_KEY=OpenAI_API_키
```

- **NOTION_TOKEN**: https://www.notion.so/profile/integrations 에서 발급
- **NOTION_DATABASE_ID**: 노션 데이터베이스 URL에서 확인 (32자리 영숫자)
- **OPENAI_API_KEY**: https://platform.openai.com/api-keys 에서 발급 (빠른 메모 기능에 필요)

### 실행
```bash
npm run dev
```
브라우저에서 http://localhost:3000 접속

### Vercel 배포 (외부 접속)
로컬이 아닌 외부(모바일 등)에서 접속하려면 Vercel에 배포하세요:
```bash
npm i -g vercel
vercel link
vercel env add NOTION_TOKEN
vercel env add NOTION_DATABASE_ID
vercel env add OPENAI_API_KEY
vercel --prod
```
배포 후 `https://프로젝트명.vercel.app` 주소로 어디서든 접속 가능합니다.

## 2. 대시보드 & 빠른 메모

**경로**: `/` (메인 페이지)

대시보드는 앱을 열면 가장 먼저 보이는 화면입니다.

### 빠른 메모 (핵심 기능)

대시보드 상단에 있는 입력창입니다. **자유롭게 아무렇게나 적으면 AI가 알아서 정리**합니다.

#### 사용법
1. 입력창에 오늘 한 일을 자유롭게 작성
2. **전송 버튼** 클릭 (또는 `Ctrl+Enter`)
3. AI가 파싱한 결과가 카드 형태로 표시됨
4. 내용을 확인하고 필요하면 수정
5. **전체 저장** 클릭 → 노션에 자동 저장

#### 입력 예시
```
오전에 A사 미팅 2시간, 오후에 로그인 버그 수정함, 내일 기획서 마감
```
→ AI가 아래처럼 3건의 업무로 분리합니다:
- A사 미팅 | 업무 | 완료 | 회의 | 2시간
- 로그인 버그 수정 | 업무 | 완료 | 버그
- 기획서 마감 | 업무 | 예정 | 기획

#### 더 많은 입력 예시
```
팀 회의 1시간
```
```
디자인 리뷰 진행중, 코드 리팩토링 완료 3시간
```
```
어제 야근해서 배포함, 오늘 오전 스프린트 회의, 오후에 신규 기능 개발 시작
```
```
저녁에 헬스장 가기, 주말에 이사 준비
```
→ 개인적인 내용은 자동으로 "개인일정"으로 분류됩니다.

#### 파싱 결과 수정
- 제목 클릭 → 직접 수정 가능
- 휴지통 아이콘 → 해당 항목 삭제
- 취소 → 처음부터 다시 입력

### 오늘의 업무

빠른 메모 아래에 **오늘 날짜의 업무만** 모아서 보여줍니다.
- 완료(✓), 진행 중(●), 예정(○) 아이콘으로 상태 구분
- 클릭하면 해당 업무 상세 페이지로 이동

### 마감 임박

"예정" 상태인 업무 중 **3일 이내 마감**인 항목을 경고 표시합니다.
- 빨간색: 오늘 마감 또는 기한 지남
- 주황색: 내일 마감
- 노란색: 2~3일 남음

마감 임박 업무가 없으면 이 섹션은 자동으로 숨겨집니다.

### 대시보드 차트

그 아래에 통계 정보가 표시됩니다:
- **요약 카드**: 총 업무 수, 총 시간, 진행 중, 완료율
- **프로젝트별 분포**: 업무/개인일정 비율 차트
- **진행상태 분포**: 예정/진행 중/완료 파이 차트
- **주간 추이**: 최근 8주간 업무량 변화 그래프
- **최근 업무**: 최근 5건의 업무 목록

## 3. 업무 목록

**경로**: `/logs` (사이드바 → 업무 목록)

등록된 모든 업무를 조회합니다. 데스크톱에서는 테이블, 모바일에서는 카드 형태로 표시됩니다.

### 필터링
상단에서 다양한 조건으로 필터링할 수 있습니다:
- **날짜 범위**: 시작일 ~ 종료일
- **프로젝트**: 업무 / 개인일정
- **진행상태**: 예정 / 진행 중 / 완료
- **태그**: 회의, 개발, 기획, 리뷰, 버그 (복수 선택 가능)
- **검색**: 제목이나 내용에서 키워드 검색

모바일에서는 **"필터" 버튼**을 눌러 필터를 펼치고 접을 수 있습니다. 활성화된 필터 수가 배지로 표시됩니다.

### CSV 다운로드
필터 아래에 있는 **"CSV 다운로드"** 버튼을 클릭하면 현재 조회된 업무 데이터를 CSV 파일로 다운로드합니다.
- 날짜, 제목, 프로젝트, 상태, 태그, 소요시간, 업무내용 컬럼 포함
- 엑셀에서 바로 열 수 있음 (한글 깨짐 없음)
- 필터를 적용한 상태에서 다운로드하면 필터링된 결과만 저장됨

### 업무 관리
- 제목 클릭 → 상세 보기
- 드롭다운 메뉴(⋯) → 수정 / 삭제

## 4. 업무 추가/수정

**경로**: `/logs/new` (사이드바 → 업무 추가)

### 입력 항목
| 항목 | 필수 | 설명 |
|------|------|------|
| 업무 제목 | O | 업무 이름 |
| 날짜 | O | 기본값: 오늘 |
| 프로젝트 | O | 업무 / 개인일정 |
| 진행상태 | O | 예정 / 진행 중 / 완료 |
| 업무내용 | - | 상세 설명 |
| 태그 | - | 회의, 개발, 기획, 리뷰, 버그 (복수 선택) |
| 소요시간 | - | 시간 단위 (0.5 단위 입력 가능) |
| 관련 링크 | - | URL |

### 프로젝트 분류 기준
- **업무**: 회사 관련 모든 업무 (회의, 개발, 보고 등)
- **개인일정**: 개인적인 일 (병원, 운동, 약속 등)

### 성과 기록 (완료 업무만)
진행상태를 **"완료"**로 선택하면 추가 입력 영역이 나타납니다:
- **성과/결과**: 이 업무의 결과물이나 성과를 기록
- **성과등급**: 상 / 중 / 하

이 정보는 나중에 **성과 관리** 페이지에서 모아볼 수 있습니다.

## 5. 캘린더

**경로**: `/calendar` (사이드바 → 캘린더)

업무 일정과 Google Calendar를 한 화면에서 확인합니다.

### 업무일지 월간 달력

노션에 등록된 업무가 날짜별로 달력에 표시됩니다.

#### 사용법
1. **좌우 화살표**로 월 이동, **"오늘"** 버튼으로 현재 월로 복귀
2. 달력의 각 날짜에 업무 제목이 표시됨 (데스크톱)
   - 모바일에서는 색상 점으로 표시 (초록=완료, 파랑=진행 중, 회색=예정)
3. **날짜를 클릭**하면 아래에 해당 날짜의 업무 목록이 상세히 표시됨
4. 업무를 클릭하면 상세 페이지로 이동

#### 색상 의미
- 초록색 배경: 완료된 업무
- 파란색 배경: 진행 중인 업무
- 회색 배경: 예정된 업무

#### 참고
- 일요일은 빨간색, 토요일은 파란색으로 표시
- 오늘 날짜는 보라색 원으로 강조
- 한 날짜에 업무가 4건 이상이면 "+N건"으로 축약 표시

### Google Calendar 연동

달력 아래에 Google Calendar "회사" 캘린더가 iframe으로 표시됩니다.

#### Google Calendar가 안 보이는 경우
1. [Google Calendar](https://calendar.google.com) 접속
2. 좌측 "다른 캘린더" 목록에서 **"회사"** 캘린더의 점 3개(⋮) 클릭
3. **"설정 및 공유"** 선택
4. **"액세스 권한"** 섹션에서 **"공개 사용 설정"** 체크
5. 저장 후 새로고침

## 6. 성과 관리

**경로**: `/achievements` (사이드바 → 성과 관리)

**인사평가 준비용 페이지**입니다. 완료된 업무 중 성과를 기록한 항목만 모아서 보여줍니다.

### 화면 구성
- **상단 카드 3개**: 성과 기록 수, 상급 성과 수, 총 투입 시간
- **성과 목록**: 각 업무의 제목, 날짜, 프로젝트, 태그, 성과등급, 성과 내용

### 활용법
1. 업무를 완료할 때마다 성과/결과를 기록
2. 인사평가 시즌에 이 페이지를 열어서 한눈에 확인
3. 상사에게 보여주거나 자기평가서에 복사하여 사용

## 7. 보고서

**경로**: `/reports` (사이드바 → 보고서)

업무 데이터를 기반으로 **보고서를 자동 생성**합니다. OpenAI API 키 없이도 동작합니다.

### 사용법
1. **빠른 선택**: "오늘", "이번 주", "이번 달" 버튼 클릭
2. 또는 시작일/종료일을 직접 입력
3. **보고서 유형** 선택: 일간 / 주간 / 월간
4. **보고서 생성** 클릭

### 생성되는 보고서 형식
```
[일일 업무 보고서] 2026-06-16 (월)

■ 완료 업무
1. A사 미팅 [업무] (2시간)
   내용: 요구사항 정리 및 일정 협의

■ 진행 중 업무
1. 로그인 기능 개발 [업무] (3시간)
   내용: OAuth 연동 작업

■ 예정 업무
1. 기획서 마감 [업무]

────────────────────────────────────────
총 업무: 3건
총 소요시간: 5시간
완료율: 33% (1/3)
```

### 내보내기
- **복사**: 클립보드에 복사 → 메신저나 이메일에 붙여넣기
- **다운로드**: .txt 파일로 저장

## 8. 통계 분석

**경로**: `/analytics` (사이드바 → 통계 분석)

전체 업무 데이터를 분석하여 **업무 패턴과 생산성**을 시각화합니다.

### 이번 주 vs 지난 주 비교
- 업무 수, 소요 시간, 완료 건수를 지난 주와 비교
- 증가/감소/동일 표시

### 차트 4종
1. **프로젝트별 시간 배분**: 업무/개인일정에 시간을 얼마나 쓰고 있는지
2. **태그별 업무 분포**: 회의/개발/기획/리뷰/버그 비율
3. **주간 완료율 추이**: 최근 8주간 완료율 변화 (상승/하락 트렌드)
4. **요일별 업무량 히트맵**: 어떤 요일에 업무가 많은지 시각화

### 활용법
- "회의 시간이 너무 많다" → 회의 줄이기
- "금요일에 업무량이 급감한다" → 금요일 집중 시간 확보
- "완료율이 떨어지고 있다" → 업무 과부하 확인

## 9. 모바일 사용법

### 접속 방법
Vercel에 배포한 경우, 모바일 브라우저에서 배포 URL을 입력하면 됩니다.
카카오톡으로 링크를 보내서 바로 접속할 수도 있습니다.

### 홈 화면에 추가 (앱처럼 사용)
PWA를 지원하므로 홈 화면에 추가하면 앱처럼 사용할 수 있습니다.

#### iPhone (Safari)
1. 사이트 접속
2. 하단 공유 버튼(□↑) 탭
3. **"홈 화면에 추가"** 선택
4. "추가" 탭

#### Android (Chrome)
1. 사이트 접속
2. 브라우저 메뉴(⋮) 탭
3. **"홈 화면에 추가"** 또는 **"앱 설치"** 선택
4. "설치" 탭

#### 설치 후
- 홈 화면에 "업무일지" 아이콘이 생김
- 탭하면 브라우저 없이 전체 화면으로 실행됨
- 일반 앱처럼 사용 가능

### 모바일 화면 특징
- **사이드바** → 좌측 상단 햄버거 메뉴(≡)로 변경
- **업무 목록** → 테이블 대신 카드 형태로 표시
- **필터** → "필터" 버튼으로 접기/펼치기 가능
- **캘린더** → 날짜에 색상 점으로 업무 표시, 탭하면 상세 보기
- **다크모드** → 햄버거 메뉴 안에 테마 변경 버튼

## 10. 노션 DB 설정

노션 데이터베이스에 아래 속성들이 있어야 합니다.

### 기본 속성
| 속성 이름 | 유형 | 비고 |
|-----------|------|------|
| 업무 | 제목 | 필수 |
| 날짜 | 날짜 | |
| 프로젝트 | 선택 | 옵션: 업무, 개인일정 |
| 진행상태 | 상태 | 옵션: 예정, 진행 중, 완료 |
| 업무내용 | 텍스트 | |
| 태그 | 다중 선택 | 옵션: 회의, 개발, 기획, 리뷰, 버그 |
| 소요시간(시간) | 숫자 | |
| 관련 링크 | URL | |

### 추가 속성 (새로 추가 필요)
| 속성 이름 | 유형 | 옵션 | 용도 |
|-----------|------|------|------|
| 성과/결과 | 텍스트 | - | 업무 결과물 기록 |
| 성과등급 | 선택 | 상, 중, 하 | 성과 중요도 평가 |
| 입력원본 | 텍스트 | - | 빠른 메모 원본 저장 |
| 입력소스 | 선택 | 웹, 빠른메모 | 어디서 입력했는지 추적 |

### 속성 추가 방법
1. 노션에서 데이터베이스 열기
2. 테이블 우측 끝 **+** 버튼 클릭
3. 속성 이름 입력 → 유형 선택
4. 선택 유형인 경우 옵션 값 추가

## 12. 문제 해결

### "빠른 메모가 안 돼요"
- `.env`에 `OPENAI_API_KEY`가 입력되어 있는지 확인
- OpenAI 계정에 크레딧(잔액)이 있는지 확인
- 서버를 재시작해보세요 (`Ctrl+C` → `npm run dev`)

### "노션에 데이터가 안 보여요"
- `.env`에 `NOTION_TOKEN`과 `NOTION_DATABASE_ID`가 올바른지 확인
- 노션 인테그레이션이 해당 데이터베이스에 연결되어 있는지 확인
  - 데이터베이스 페이지 우측 상단 `...` → 연결 → 인테그레이션 선택

### "성과 기록 영역이 안 나타나요"
- 진행상태를 **"완료"**로 선택해야 성과 기록 영역이 표시됩니다

### "Google Calendar가 안 보여요"
- Google Calendar에서 "회사" 캘린더의 **공개 사용 설정**이 되어 있는지 확인
- 설정 방법: Google Calendar → 설정 → 회사 캘린더 → 액세스 권한 → 공개 사용 설정 체크

### "모바일에서 화면이 이상해요"
- 브라우저 캐시를 삭제하고 새로고침해보세요
- PWA로 설치한 경우 삭제 후 다시 설치해보세요

### "CSV 파일이 엑셀에서 깨져요"
- UTF-8 BOM이 포함되어 있어 일반적으로 깨지지 않습니다
- 그래도 깨지면: 엑셀 → 데이터 → 텍스트에서 가져오기 → UTF-8 선택

### "빌드 에러가 나요"
```bash
npm install
npm run build
```
에러 메시지를 확인하세요.

### 서버 포트 충돌
이미 3000번 포트를 사용 중이면:
```bash
npx next dev -p 3001
```
