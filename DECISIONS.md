## 2026-07-18 앱 방향 진단과 AI 전략

### 사용 실태 진단

실사용 확인 결과 쓰는 축과 안 쓰는 축이 명확히 갈림.

- **주 1회 이상 사용**: 대시보드·빠른 메모(입력), 조회 뷰(목록·칸반·캘린더·주간 리뷰), 운영(HR·급여·트랙)
- **거의 안 씀**: 통계 분석, 성과 관리, 보고서, 법인 통합 뷰 — 분석·보고 4탭 전부
- **체감 가치**: 기록 습관이 생겼고, 잊을 일을 안 잊게 됨

**결론: 이 앱의 정체성은 분석 도구가 아니라 "기억 보조 + 운영 장부".**
분석·보고 축은 수요 없이 만들어진 것 (실패가 아니라 학습).

### 확정한 원칙

1. **새 기능 추가 기준**: "지난주에 이게 없어서 실제로 불편했던 순간이 있었나."
   예스가 아니면 만들지 않는다.
2. **AI 방향은 pull → push**: 버튼 눌러 요약받는 방식보다 AI가 먼저 말 거는 방식에 투자.
   1순위 아침 브리핑(2026-07-18 구현, `d0df0b9`), 2순위 주간 리뷰 AI 코치.
3. **하지 말 것**: 데이터 수십 건 수준에서 패턴 분석·업무량 예측·우선순위 자동 추천.
   데이터 수백 건 쌓인 뒤에나 의미 있음.
4. **보류**: 안 쓰는 분석·보고 4탭을 사이드바 접이식 그룹으로 내리기 (제안됨, 미실행).

### 도구 선택 기준 (큰 프로젝트 대비)

법인 설립·수출·해외 오픈 같은 큰 프로젝트는 하나의 기능이 아니라 성격이 다른 업무 묶음:

| 업무 성격 | 도구 |
|-----------|------|
| 진행 관리 (체크리스트·마감·단계) | 이 앱의 트랙 기능 |
| 업체 소통 | 카톡·메일 그대로, **결과만** 앱에 기록 (카톡 파싱 활용) |
| 구인·구직 | 트랙 업무 항목 → 채용 후 HR 기능 |
| 매출·BEP (숫자 반복 축적) | **Notion 아님.** 초기엔 엑셀/구글시트, 다점포 되면 Supabase |

기준: 텍스트 중심 저볼륨 기록 → Notion. 숫자 중심 반복 축적 계산 → 시트/DB.

### 요아정 재팬 오픈 관련

- 오픈 실무는 담당 직원이 엑셀로 관리 (체크리스트 52항목 + 일일 매출표). 잘 하고 있어
  도구 교체 불필요. 본인은 **참관·학습 포지션** — 관여 없음.
- **오픈 관리 기능을 앱에 만들지 않기로 확정** (이전에 세웠던 체크리스트·비용 DB 계획 폐기).
- 배우는 방법: 체크리스트의 순서와 이유(왜 비자→법인통장→인허가 순인지)를 눈으로 따라가고,
  모르는 용어는 빠른 메모로 기록. 프로젝트 끝나면 완성된 체크리스트가 교과서로 남음.
- 나중에 업무를 맡게 되면: 도구를 바꾸지 말고 엑셀 승계 + 파일 폴더 정리 + 인수인계
  (업체 연락처, 계약서 원본, 계정 정보, 미결 상태)에 집중.
